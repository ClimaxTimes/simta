Tabel 4.1 Spesifikasi Koleksi Users

No Nama Field Tipe Data Deskripsi
1 _id ObjectId Pengenal unik yang dibangkitkan secara otomatis oleh basis data MongoDB.
2 nim_nip String Nomor Induk Mahasiswa atau Nomor Induk Pegawai sebagai identitas unik pengguna.
3 password String Kredensial keamanan pengguna yang telah terenkripsi menggunakan algoritma bcrypt.
4 name String Nama lengkap pengguna sesuai dengan data kependudukan.
5 email String Alamat surat elektronik pengguna untuk keperluan komunikasi.
6 role String Peran pengguna dalam sistem dengan nilai enum: mahasiswa, dosen, atau admin.
7 prodi String Program studi tempat mahasiswa terdaftar (khusus role mahasiswa).
8 semester String Semester aktif mahasiswa saat ini (khusus role mahasiswa).
9 judulTA String Judul Tugas Akhir atau Skripsi yang sedang dikerjakan mahasiswa.
10 currentProgress String Indikator tahapan bab terakhir yang diselesaikan dengan nilai: BAB I hingga Selesai.
11 dospem_1 ObjectId Referensi ID ke data Dosen sebagai Pembimbing Utama (Pembimbing 1).
12 dospem_2 ObjectId Referensi ID ke data Dosen sebagai Pembimbing Pendamping (Pembimbing 2).
13 status String Status keaktifan akun dengan nilai enum: aktif atau nonaktif.
14 whatsapp String Nomor telepon WhatsApp untuk keperluan komunikasi langsung.
15 createdAt Date Stempel waktu penciptaan dokumen yang dibangkitkan secara otomatis.
16 updatedAt Date Stempel waktu pembaruan terakhir dokumen yang dibangkitkan secara otomatis.

Tabel 4.2 Spesifikasi Koleksi Bimbingans

No Nama Field Tipe Data Deskripsi
1 _id ObjectId Pengenal unik yang dibangkitkan secara otomatis oleh basis data MongoDB.
2 mahasiswa ObjectId Referensi ID ke koleksi Users yang mengidentifikasi mahasiswa pengunggah.
3 dosen ObjectId Referensi ID ke koleksi Users yang mengidentifikasi dosen penerima bimbingan.
4 dosenType String Klasifikasi dosen pembimbing dengan nilai enum: dospem_1 atau dospem_2.
5 version String Penanda versi dokumen bimbingan dengan format penomoran V1, V2, V3, dan seterusnya.
6 judul String Judul atau subjek bimbingan yang diajukan oleh mahasiswa.
7 catatan String Catatan atau pesan tambahan dari mahasiswa kepada dosen pembimbing.
8 fileName String Nama berkas yang tersimpan pada sistem penyimpanan server.
9 filePath String Lokasi absolut berkas dokumen pada sistem penyimpanan server.
10 fileSize String Ukuran berkas dokumen dalam satuan byte.
11 status String Status peninjauan bimbingan dengan nilai enum: menunggu, revisi, acc, atau lanjut_bab.
12 feedback String Umpan balik atau komentar yang diberikan oleh dosen pembimbing.
13 feedbackDate Date Stempel waktu pemberian umpan balik oleh dosen pembimbing.
14 feedbackFile String Lokasi berkas lampiran yang disertakan oleh dosen sebagai umpan balik.
15 createdAt Date Stempel waktu penciptaan dokumen yang dibangkitkan secara otomatis.
16 updatedAt Date Stempel waktu pembaruan terakhir dokumen yang dibangkitkan secara otomatis.

Tabel 4.3 Spesifikasi Koleksi Replies

No Nama Field Tipe Data Deskripsi
1 _id ObjectId Pengenal unik yang dibangkitkan secara otomatis oleh basis data MongoDB.
2 bimbingan ObjectId Referensi ID ke koleksi Bimbingans sebagai induk percakapan.
3 sender ObjectId Referensi ID ke koleksi Users yang mengidentifikasi pengirim pesan.
4 senderRole String Peran pengirim pesan dengan nilai enum: mahasiswa atau dosen.
5 message String Isi pesan atau komentar yang disampaikan dalam diskusi.
6 createdAt Date Stempel waktu penciptaan dokumen yang dibangkitkan secara otomatis.
7 updatedAt Date Stempel waktu pembaruan terakhir dokumen yang dibangkitkan secara otomatis.

Tabel 4.4 Spesifikasi Koleksi Jadwals

No Nama Field Tipe Data Deskripsi
1 _id ObjectId Pengenal unik yang dibangkitkan secara otomatis oleh basis data MongoDB.
2 mahasiswa ObjectId Referensi ID ke koleksi Users yang mengidentifikasi mahasiswa peserta sidang.
3 jenisJadwal String Klasifikasi jenis sidang dengan nilai enum: sidang_proposal atau sidang_skripsi.
4 tanggal Date Tanggal pelaksanaan sidang yang dijadwalkan.
5 waktuMulai String Jam dimulainya sidang dalam format HH:MM.
6 waktuSelesai String Jam berakhirnya sidang dalam format HH:MM.
7 ruangan String Lokasi ruangan tempat pelaksanaan sidang.
8 penguji Array[ObjectId] Daftar referensi ID ke koleksi Users yang berperan sebagai dosen penguji.
9 status String Status pelaksanaan sidang dengan nilai enum: dijadwalkan, berlangsung, selesai, atau dibatalkan.
10 hasil String Hasil penilaian sidang dengan nilai enum: lulus, lulus_revisi, atau tidak_lulus.
11 nilaiSidang Number Nilai numerik hasil sidang dalam rentang 0 hingga 100.
12 catatan String Catatan tambahan terkait pelaksanaan atau hasil sidang.
13 createdBy ObjectId Referensi ID ke koleksi Users yang mengidentifikasi administrator pembuat jadwal.
14 createdAt Date Stempel waktu penciptaan dokumen yang dibangkitkan secara otomatis.
15 updatedAt Date Stempel waktu pembaruan terakhir dokumen yang dibangkitkan secara otomatis.

Tabel 4.5 Relasi Antar Koleksi

Koleksi Asal Koleksi Tujuan Jenis Relasi Keterangan
Users Users One-to-One Mahasiswa memiliki referensi ke Dosen Pembimbing 1 dan 2.
Users Bimbingans One-to-Many Satu mahasiswa dapat mengunggah banyak dokumen bimbingan.
Users Jadwals One-to-Many Satu mahasiswa dapat memiliki banyak jadwal sidang.
Users Jadwals Many-to-Many Banyak dosen dapat menjadi penguji di banyak jadwal sidang.
Bimbingans Replies One-to-Many Satu bimbingan dapat memiliki banyak pesan diskusi.
Users Replies One-to-Many Satu pengguna dapat mengirim banyak pesan diskusi.
